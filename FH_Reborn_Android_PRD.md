## 1. 项目概述

### 1.1 产品定位

一款面向本地音乐收藏爱好者的**高品质本地音乐播放器**，覆盖 Android（移动主力）、Linux（桌面）、Windows（桌面）三平台。各平台采用独立原生技术栈开发，追求极致的平台原生体验与设计一致性。

### 1.2 核心设计理念

- **视觉优先**：流体动态背景、Fluent Design 材质（云母/亚克力）、文字与图标的混合光效
- **播放体验**：类 Salt Player / Apple Music 的沉浸式播放界面，逐字歌词、弹簧动画
- **专业功能**：多格式歌词支持、播放统计、灵感记录、音频属性查看
- **跨平台一致**：统一的功能清单与设计规范，各平台按自身技术栈独立实现

### 1.3 开发策略

| 阶段 | 目标 | 内容 |
|------|------|------|
| **第一阶段（MVP）** | 可播放的媒体库 | 媒体库扫描 + 基础播放 + 基础 UI + LRC 歌词 |
| **第二阶段（增强）** | 体验提升 | FFmpeg 补充扫描 + TTML + 流体背景 + 律动 + 歌单系统 |
| **第三阶段（完善）** | 功能完备 | 数据持久化 + 导出 + 灵感记录 + 主题/模糊材质系统 |

## 2. 用户画像

### 2.1 核心用户角色

**音乐发烧友**
- 身份：拥有大量本地音乐收藏（无损格式为主）
- 核心目标：在移动和桌面端享受高品质的本地音乐播放体验
- 主要痛点：现有播放器不支持多格式歌词、缺乏播放统计、UI 不够美观

**多平台用户**
- 身份：同时使用 Android 手机和 Linux/Windows 电脑
- 核心目标：在不同设备上获得一致且优质的音乐体验
- 主要痛点：各平台播放器体验割裂，歌单难以迁移

**歌词爱好者**
- 身份：喜欢边看歌词边听歌，关注逐字歌词和翻译
- 核心目标：精确的歌词同步、多种歌词格式支持、美观的歌词展示
- 主要痛点：很多播放器仅支持基础 LRC，不支持 TTML 逐字歌词

### 4.1 媒体库模块（F-001 ~ F-006, M-001 ~ M-006）

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-001 | MediaStore 快速索引 | 利用系统 MediaStore API 秒级扫描已知音频格式（MP3、M4A、FLAC 等） | Must |
| F-002 | FFmpeg 补充扫描 | 对系统不支持的格式（APE、DSD 等）通过 FFmpeg 遍历目录解析元数据 | Must |
| F-003 | 隐藏文件夹 | 跳过 .nomedia 目录，支持用户自定义黑名单路径，可切换显示/隐藏 | Must |
| F-004 | 目录浏览 | 按文件系统树状结构浏览音频文件，不依赖扫描结果 | Must |
| F-005 | 模糊搜索 | 支持按歌曲名、艺术家、专辑名本地模糊搜索 | Must |
| F-006 | 拼音首字母搜索 | 支持拼音首字母快速检索中文歌曲 | Should |
| M-001 | 启动时自动更新 | 每次启动时自动检测媒体库变更并增量更新 | Must |

#### 扫描策略（Android）— 已确认决策

**扫描范围：用户指定目录（不全盘扫描）**
- 首次启动：引导用户选择 1 个或多个音乐目录（默认推荐 `/Music`、`/Download`）
- 仅扫描用户指定目录及其子目录，不遍历全盘
- 用户后续可在设置中新增/移除扫描目录
- 隐藏文件夹过滤：扫描时实时检查 .nomedia 文件和用户黑名单

**双轨扫描流程：**
```
启动时自动更新 ──→ MediaStore 快速索引（指定目录范围）──→ 秒级显示
                              │
                              └──→ 后台 FFmpeg 补充扫描（仅指定目录）
                                         │
                                         └──→ 合并入库 → Flow 推送 UI
```

**FFmpeg 集成方式：静态库集成**
- 通过 `com.arthenica:ffmpeg-kit-min` 或类似静态包集成
- 随 APK 打包，首次启动即可用，无需额外下载
- 缺点：APK 体积增加约 15-30MB（视 ABI 数量而定）
- 仅引入所需的解码器子集（APE、DSD、WAVPACK），裁剪无用模块以减小体积

**启动时自动更新（M-001）**：
- 每次应用启动时自动检测媒体库变更
- 监听方式：MediaStore ContentObserver + 文件修改时间对比
- 增量更新：仅扫描新增/修改/删除的文件，不重复扫描已有文件
- 后台执行：不阻塞用户操作，扫描完成后静默刷新列表

**性能优化**：kotlinx.coroutines + Flow 实时更新 UI

**支持音频格式**：
MP3、FLAC、AAC、OGG、WAV、M4A、OPUS、APE、DSD(DSF/DFF)

#### 封面/缩略图缓存策略（待确认）

三种方案可选：

| 方案 | 说明 | 优点 | 缺点 | 推荐 |
|------|------|------|------|------|
| **A. 懒加载 + 内存缓存** | 仅在需要显示时从音频文件提取封面，内存 LRU 缓存 | 无额外磁盘占用，简单 | 列表滚动时反复解码，耗电 | — |
| **B. 磁盘缓存（缩略图）** | 扫描时提取封面保存为缩略图文件（如 `/cache/covers/`），显示时加载缩略图 | 列表流畅，不反复解码 | 首次扫描耗时增加，占用磁盘空间（约 50-200MB/千首） | **推荐** |
| **C. 混合策略** | 磁盘缓存缩略图 + 内存 LRU + 播放页按需加载原图 | 最佳体验，列表流畅且播放页清晰 | 实现复杂，磁盘+内存双重占用 | — |

**建议**：方案 B 或 C。千首歌曲的缩略图约 50-200MB，现代设备可接受。扫描时异步生成缩略图（128x128px for 列表，512x512px for 播放页）。

#### 播放控制（P-001 ~ P-003）— 已确认决策

**基础控制交互**：

| 操作 | 触发方式 | 行为 |
|------|---------|------|
| 播放/暂停 | 播放按钮 / 耳机单按 | 切换播放/暂停状态 |
| 上一首 | 上一首按钮点击 / 耳机三按 | 3秒内从头播放，超过3秒跳到开头 |
| 下一首 | 下一首按钮点击 / 耳机双按 | 切换到下一首 |
| **长按快退** | **长按上一首按钮** | **持续快退（步进-5s/s）** |
| **长按快进** | **长按下一首按钮** | **持续快进（步进+5s/s）** |
| **指哪打哪** | **点击进度条任意位置** | **直接跳转到对应时间点，无需拖拽** |

**耳机按键控制（已确认）**：
- 单按：播放/暂停
- 双按：下一首
- 三按：上一首

**播放进度保存（已确认）**：每 5 秒自动保存当前播放位置到数据库

#### 播放模式（P-004）— 四种标准模式

四种模式**独立切换**，互不影响：

| 模式 | 图标 | 行为 |
|------|------|------|
| **顺序播放** | ➡️ | 播完队列最后一首后停止 |
| **列表循环** | 🔁 | 播完队列最后一首后回到第一首继续 |
| **单曲循环** | 🔂 | 当前单曲无限重复播放 |
| **随机播放** | 🔀 | 真随机打乱队列顺序播放 |

**随机模式实现**：
- 开启时生成乱序队列副本，不修改原始队列顺序
- 同一首歌在 10 首内不重复出现
- 关闭随机时恢复原始队列顺序，回到当前歌曲的原始位置
- 下一首/上一首在随机队列中导航

#### 播放队列（P-005）— 已确认决策

**UI 布局：与播放器页面一体**
- 不是 BottomSheet，而是播放页的一部分（如侧滑进入或同页面切换 Tab）
- 显示当前队列列表，高亮当前播放项

**队列操作**：
- ✅ 拖拽排序
- ✅ 删除单项（侧滑删除或长按菜单）
- ✅ **添加到下一首播放**（从其他列表选择歌曲时的操作）
- ❌ 不直接提供"添加歌曲"到队列（队列由当前播放上下文决定）

**队列来源规则**：
- 点击歌曲播放时，以该歌曲所在的列表（全部歌曲/专辑/歌单/搜索结果）为队列
- 从其他页面选择"添加到下一首播放"时，仅将该歌曲插入到当前播放位置的下一首

#### 音频焦点（P-008）— 已确认决策

**默认行为**：

| 场景 | 默认行为 | 可配置项 |
|------|---------|---------|
| 来电 | 暂停，通话结束不恢复 | 可改为"通话结束自动恢复" |
| 导航语音 | 降低音量至 20%，语音结束自动恢复 | 可改为"暂停"或"不处理" |
| 其他 App 音乐 | 暂停，对方结束后自动恢复 | 可改为"不处理" |
| 蓝牙耳机断开 | 暂停 | 可改为"不处理"（继续扬声器播放） |

**设置入口**：设置 → 音频 → 音频焦点 → 各场景独立开关

#### 后台播放 + MediaSession（P-009）

- 使用 `MediaLibraryService`（Media3）实现后台播放
- 播放时前台通知，暂停 5 分钟后自动移除通知
- 媒体通知按钮：上一首 / 播放暂停 / 下一首 / 收藏（展开时）/ 关闭
- Android 13+ 通知支持进度条和 Seek

> **Android 播放器模块 — 已确认决策**：长按上一首/下一首=快进快退 | 进度=指哪打哪 | 耳机=单按暂停/双按下首/三按上首 | 播放模式=四种标准 | 队列=播放页一体 | 手势=封面+MiniBar左滑下首右滑上首 | 长按封面=查看/分享/保存 | 双击封面=暂不 | 简洁模式=菜单手动(封面+信息+进度) | 音频焦点=可配置 | 进度保存=5s

#### Reveal 按压光效（U-005）— 需自行实现

**状态**：无现成 Compose 实现，需自定义 `Indication` + `Modifier.pointerInput`

**实现方案**：
```kotlin
// 自定义 Indication
@Composable
fun rememberRevealIndication(
    color: Color = FluentTheme.colors.accent,
    radius: Dp = 120.dp
): Indication {
    return remember {
        object : Indication {
            @Composable
            override fun rememberUpdatedInstance(
                interactionSource: InteractionSource
            ): IndicationInstance {
                val animatedPos = Animatable(Offset.Zero, Offset.VectorConverter)
                val animatedAlpha = Animatable(0f)
                
                LaunchedEffect(interactionSource) {
                    interactionSource.interactions.collect { interaction ->
                        when (interaction) {
                            is PressInteraction.Press -> {
                                animatedPos.snapTo(interaction.pressPosition)
                                animatedAlpha.animateTo(0.15f, spring())
                            }
                            is PressInteraction.Release,
                            is PressInteraction.Cancel -> {
                                animatedAlpha.animateTo(0f, tween(300))
                            }
                        }
                    }
                }
                
                return object : IndicationInstance {
                    override fun ContentDrawScope.drawIndication() {
                        drawContent()
                        // 径向光晕
                        drawCircle(
                            color = color.copy(alpha = animatedAlpha.value),
                            radius = radius.toPx(),
                            center = animatedPos.value
                        )
                    }
                }
            }
        }
    }
}
```

**与流体律动背景配合**：
- Reveal 光晕颜色 = 强调色 + 当前流体背景色混合
- 按压时光晕与流体色彩交融，产生"触摸水面"般的视觉效果
- 这是视觉决胜点：流体律动背景 + Reveal 光效的组合效果

#### 顶部滚动渐变模糊（U-009）— Haze 效果应用

**效果描述**：
- 内容区域顶部有一个 48-64dp 高度的渐变模糊层
- 当列表向上滚动时，顶部的内容逐渐模糊并淡出
- 营造"内容从模糊中浮现"的层次感，类似 iOS 的滚动模糊效果

**技术实现（Haze）**：
```kotlin
// 使用 Haze 库实现顶部渐变模糊
Box(modifier = Modifier.fillMaxSize()) {
    // 内容列表（顶部和底部都留 insets 空）
    LazyColumn(
        contentPadding = PaddingValues(
            top = 64.dp,  // 顶部留空防止被状态栏遮挡
            bottom = bottomInsetHeight  // 底部留空防止被三栏遮挡
        )
    ) { /* ... */ }
    
    // 顶部渐变模糊层（固定在内容上方）
    Box(
        modifier = Modifier
            .fillMaxWidth()
            .height(64.dp)
            .align(Alignment.TopCenter)
            .haze(
                state = hazeState,
                style = HazeStyle(
                    tint = Brush.verticalGradient(
                        0f to Color.Transparent,
                        1f to Color.Black.copy(alpha = 0.3f)
                    ),
                    blurRadius = 20.dp
                )
            )
    )
}
```

- 使用 **Haze** 库：`Modifier.haze()` + `HazeStyle` 渐变模糊
- 渐变方向：顶部透明 → 向下逐渐模糊
- 与内容滚动联动：滚动速度影响模糊强度（快速滚动时模糊加深）

| F-063 | Reveal 按压光效 | 跟随指针位置的 Reveal Highlight 光效，用于按钮/列表/控件的 pressed 状态，搭配流体背景形成视觉决胜点 | Should |

**BlendMode 发光效果说明**：
文字/图标的"发光"不是简单的阴影，而是使用 Android BlendMode（混合模式）实现的像素级光效叠加：
- **Overlay**：根据底图亮度自动选择 multiply 或 screen，产生"镶嵌在背景上"的融合感
- **Plus Lighter**：`min(src + dst, 1.0)`，将发光层与背景直接相加，产生霓虹灯般的明亮溢出效果
- 应用场景：歌曲标题在流体背景上的可读性、歌词逐字高亮的发光、播放按钮点击时的闪光

**技术实现路线**：
- 优先用阴影模糊模拟发光（全版本兼容，第一阶段采用）
- Android 12+ 可用 `BlendMode.PLUS` / `BlendMode.OVERLAY` 原生实现
- Android 12+ 可用 `RuntimeShader` / `AGSL` 做更精确的控制

**Reveal 按压光效技术方案**：
- 使用 Compose `Indication` API + `Modifier.pointerInput` 实现
- 监听 `PressInteraction.Press` 获取触点坐标
- 在触点位置绘制径向光晕，颜色跟随 Fluent 强调色
- 光晕淡入淡出使用 `Animatable` + `spring()` 动画
- 与流体背景配合：按压时光晕与背景流体色彩交融，形成视觉决胜点

### 5.1 设计体系

| 平台 | 设计框架 | 材质效果 | 交互反馈 |
|------|---------|---------|---------|
| **Android** | Compose Fluent | Mica/Acrylic via Fluent 组件 | **Reveal Highlight 光效**（pressed 状态） |

| **Windows** | WinUI 3 原生 Fluent | 原生 Mica/Acrylic | **原生 Reveal** |

- **模糊材质**：大量应用 Haze/Acrylic 模糊效果
- **页面边缘渐变模糊**：标题栏/底栏使用 Haze 渐变模糊
- **歌词页面**：顶部和底部渐变淡出，中间歌词清晰
- **自定义背景**：播放页和 App 其他页面均支持自选图片背景 + 前景色调整
- **交互反馈**：Reveal Highlight 光效用于按钮/列表/控件的 pressed 状态，搭配流体背景形成视觉决胜点

### 5.3 播放页视觉层次

1. **底层**：自定义背景图片或流体动态背景
2. **中层**：律动可视化效果（频谱/波形）
3. **上层**：专辑封面、歌曲信息、控制按钮
4. **文字效果**：标题/歌词可能应用发光效果

## 9. 风险与约束

### 9.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| Compose Fluent 不稳定 | Android UI 风格无法统一 | 中 | 引入 Haze 作为模糊效果兜底，封装一层便于切换 |
| FFmpeg 集成复杂度 | 扫描性能问题、包体积增大 | 中 | 后台线程解析，按需引入 FFmpeg 模块 |
| 桌面端音频可视化 | Linux/Windows 获取 PCM 数据方案不同 | 高 | 各平台独立实现音频处理器 |
| 跨平台歌单兼容 | M3U 路径格式差异 | 低 | JSON 格式作为主力导出格式 |

### 9.2 约束条件

- **Android 最低版本**：Android 12（API 31），支持 RenderEffect.blurBitmap()、Material You 动态取色等现代 API
- **Linux 发行版**：需适配不同包管理器和 GStreamer 版本
- **Windows 部署**：WinUI 3 Runtime 版本地狱，开发阶段用 Self-Contained
- **开发人力**：单人/小型团队，三平台并行开发周期长
